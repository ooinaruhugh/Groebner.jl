    Utilities

    Assertions

    To enable/disable all @invariant macro in Groebner.jl, change Groebner.invariants_enabled() to true/false.

    Groebner.@invariant
    macro

    @invariant expr

    Check that expr evaluates to true at runtime. If not, throw an AssertionError.

    Examples

    @invariant 2 > 1
@invariant 1 > 2  # throws!

    Logging

    To enable/disable all @log macro in Groebner.jl, change Groebner.logging_enabled() to true/false.

    For example,

    using Groebner, AbstractAlgebra
_, (x,y,z) = GF(2^31-1)["x","y","z"]
groebner([x*y + z, x*z + y], loglevel=-3)
    3-element Vector{AbstractAlgebra.Generic.MPoly{AbstractAlgebra.GFElem{Int64}}}:
 y^2 + 2147483646*z^2
 x*z + y
 x*y + z

    Groebner.@log
    macro

    @log expr
@log level=N expr

    Logs a record with expr as a message. Allows to specify the logging level with level=N.

    Examples

    @log "Hello, world!"
@log level=1 "Hello, world!"

    Collecting statistics and measuring performance

    All functions in the interface of Groebner.jl have keyword argument statistics. This keyword argument can be set to either of: :no, :timings, and :all.

    To enable/disable all @log macro in Groebner.jl, change Groebner.logging_enabled() to true/false.

    For example,

    using Groebner, AbstractAlgebra
_, (x,y,z) = GF(2^31-1)["x","y","z"]
groebner([x*y + z, x*z + y], statistics=:timings)
    AssertionError: Not recognized keyword: statistics
